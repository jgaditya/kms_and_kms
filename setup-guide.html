Living Heritage Project Setup Guide

JSON Parse Error Resolution

If you're getting a JSON parse error when running npm init -y, follow these steps:

Step 1: Clear npm Cache

Open your terminal in VS Code and run:

npm cache clean --force

Step 2: Verify npm Configuration

Check your npm configuration with:

npm config list

Look for any invalid JSON configurations.

Step 3: Initialize npm Properly

Try initializing with a scope or different approach:

npm init -y --scope=yourusername

Or manually create a package.json file:

{
  "name": "living-heritage",
  "version": "1.0.0",
  "description": "A platform for exploring and selling cultural experiences",
  "main": "app.js",
  "scripts": {
    "start": "node app.js",
    "dev": "nodemon app.js"
  },
  "keywords": ["heritage", "culture", "tourism"],
  "author": "Your Name",
  "license": "MIT"
}

Step 4: Install Dependencies

After creating package.json, install required dependencies:

npm install express mongoose bcryptjs jsonwebtoken cors dotenv

Install development dependencies:

npm install -D nodemon

Step 5: Create Environment File

Create a .env file in your root directory:

MONGODB_URI=mongodb://localhost:27017/living-heritage
JWT_SECRET=your_super_secret_jwt_key_here
PORT=5000

Step 6: Set Up Your Main App File

Create app.js with the following content:

const express = require('express');
const mongoose = require('mongoose');
const cors = require('cors');
require('dotenv').config();

// Import routes
const authRoutes = require('./routes/auth');
const explorerRoutes = require('./routes/explorer');
const merchantRoutes = require('./routes/merchant');

const app = express();

// Middleware
app.use(cors());
app.use(express.json());
app.use(express.static('public'));

// Routes
app.use('/api/auth', authRoutes);
app.use('/api', explorerRoutes);
app.use('/api', merchantRoutes);

// Database connection
mongoose.connect(process.env.MONGODB_URI, {
  useNewUrlParser: true,
  useUnifiedTopology: true,
})
.then(() => console.log('MongoDB connected successfully'))
.catch(err => console.log('MongoDB connection error:', err));

const PORT = process.env.PORT || 5000;

app.listen(PORT, () => {
  console.log(`Server is running on port ${PORT}`);
});

Final Step: Run Your Application

Start your development server with:

npm run dev

If everything is set up correctly, you should see:

Server is running on port 5000
MongoDB connected successfully

Troubleshooting

If you're still having issues:

  • Make sure you have Node.js and npm installed correctly
  • Check your internet connection
  • Try updating npm: npm install -g npm@latest
  • Verify there are no syntax errors in any existing package.json file
